Recommended Architecture

Static site. No database. No server. Just files.

Why NOT WordPress / React / Custom?

WordPress: security burden, ongoing maintenance. React/Next.js: premature — build static first, enhance later. The Chairman's standard: sovereign, zero-cost, version-controlled.

Anti-Patterns — What We Don't Do

  • Don't build a database. Static files are enough. Databases need maintenance.
  • Don't embed PDFs in the page. Link to them. Let users download.
  • Don't add comments/discussion. This is an archive, not social media.
  • Don't make it a SPA. Server-render everything. Speed > interactivity.
  • Don't gate any documents. No login, no email, no paywall.
  • Don't editorialize. Present documents neutrally. Let the content speak.
  • Don't ignore non-English sources. Flag them for Phase 2 translation.
